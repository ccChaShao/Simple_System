# 通用红点系统

通用结构，仅包含必要逻辑，不包含具体项目相关内容。

## ✨ 主要功能

- **功能1:** 红点样式可配置（自定义样式、锚点预设）；
- **功能2:** 对预制体资源无侵入性；
- **功能3:** 开发，使用成本低；
- **功能4:** 拓展性高，适用各种项目开发环境；
- **功能5:** 红点UI实例用对象池进行缓存（未完全接入）；

## 🚀 代码结构描述

系统分为数据层、表现层、管理层；

1. 数据层（RedDotNode.cs）  
仅提供数据相关的存储、处理接口；例如红点路径（key），父节点，子节点字典，更新回调。  


2. 表现层（RedDotView.cs）  
仅处理统筹红点数据和表现相关的内容，可序列化类；例如数据更新时，表现层的回调注册及反注册，红点类型的存储。  


3. 管理层（RedDotModule.cs）  
用于统筹红点数据相关的更新，对数据进行按需统一更新，提供数据相关的增删改查、数据更新事件的注册反注册接口；同时内部维护一个更新缓存表，为了统一更新时机，会将有diff或者被diff的数据进行cache，在帧末统一更新。  


4. 项目接入脚本（Manager.cs,RedDotItem.cs）  
仅需要进行管理层的初始化，以及处理editor控制面板，接入项目的资源加载环境即可。

## 🛠️ 运作流程描述
项目接入：

1. 提供红点对应的资源（预制体、图标Sprite）；


2. 编写项目对应的整合脚本（unity：mono...），同时接入项目提供的资源加载接口；   
   
开发流程：
1. 在合适的时机进行红点数据的初始化以及更新（进入世界：初始化，交互后：更新...）；


2. 在UI加载完成的生周期内进行红点注册（提供：路径、挂载点）；
